# Çoklu Dil (i18n) — Rehber

Uygulama `next-intl` + App Router `[locale]` yapısı ile 4 dili destekler:
**tr** (varsayılan), **es**, **it**, **pt**. Bu belge, henüz çevrilmemiş
sayfaları taşımak için izlenecek deseni anlatır.

## Kurulum (bitti)

- `src/i18n/routing.ts` — diller, varsayılan dil, `localePrefix: "as-needed"`
  (tr prefix almaz: `/dashboard`; diğerleri alır: `/es/dashboard`).
- `src/i18n/request.ts` — istek başına mesaj kataloğunu yükler.
- `src/middleware.ts` — auth guard + dil yönlendirmesi BİRLEŞİK. Dil prefix'i
  auth kararından ÖNCE soyulur; aksi halde `/es/dashboard` korumalı sayılmaz ve
  auth'suz açılırdı. Bu dosyayı değiştirirken dikkat.
- `messages/{tr,es,it,pt}.json` — çeviri katalogları.
- `src/lib/format.ts` — dil-farkında biçimlendirme. **Bu dosyanın dışında
  `toLocaleString` KULLANMAYIN.**
- `src/hooks/useTenantCurrency.ts` — tenant'ın para birimi (`GET /policy`).

## Referans: `src/app/[locale]/dashboard/page.tsx`

Tam çevrilmiş tek sayfa. Yeni bir sayfayı taşırken bunu örnek al.

## Bir sayfayı taşıma adımları

1. **Namespace aç**: `messages/*.json` içinde sayfaya bir üst anahtar ekle
   (ör. `customers`). Anahtarları 4 katalogda da tut.
2. **Client component'te bağlan**:
   ```tsx
   import { useTranslations, useLocale } from "next-intl";
   import type { Locale } from "@/i18n/routing";
   const t = useTranslations("customers");
   const locale = useLocale() as Locale;
   ```
3. **Sabit metinleri değiştir**: `"Müşteriler"` → `{t("title")}`.
4. **Sayı/tarih/para**: elle yazılmış `toLocaleString("tr-TR")` ve `₺` yerine
   `format.ts` fonksiyonları:
   - `formatKg(value, locale)` — yağ/zeytin kg
   - `formatNumber(value, locale, opts)` — genel sayı, randıman
   - `formatDate(value, locale)` / `formatDateTime(value, locale)`
   - `formatPercent(value, locale)`
   - `formatCurrency(value, currency, locale)` — **currency parametre**,
     `useTenantCurrency()`'den gelir. ₺/TRY SABİTLEMEK YASAK.
5. **Link'ler**: `next/link` yerine `@/i18n/navigation`'dan `Link`/`useRouter`
   kullan; dil prefix'ini otomatik korurlar.

### Özel durumlar

- **`app/[locale]/production/page.tsx` Excel dışa aktarımı**: kolon başlıkları
  (`"Parti No"`, `"Müşteri"` ...) ve `"Evet"/"Hayır"` değerleri Türkçe object
  key'leri olarak yazılı. Bunları çevirmek `lib/export.ts`'e giden veri yapısını
  değiştirmeyi gerektirir, sadece etiket değişimi değil.
- **`app/[locale]/legal/*` (KVKK/şartlar)**: ÇEVİRMEYİN. Bunlar Türkiye'ye özgü
  (KVKK) hukuki metin. ES/IT/PT için GDPR'a göre her yargı alanında ayrı ayrı
  bir hukukçu tarafından yazılmalı — çeviri değil, yeniden kaleme alma işi.
- **Backend hata mesajları**: servisler Türkçe `throw` ediyor (ör.
  `"Yetersiz yağ bakiyesi."`). Kullanıcıya görünüyorlar. İdeali: backend'in
  kararlı hata KODLARI dönmesi, çevirinin frontend'de yapılması. Bu ayrı bir iş.

## Katalog anahtar kuralı

Namespace = ekran/alan (`common`, `dashboard`, `customers`). Anahtar =
kebab-case değil, camelCase kısa isim (`common.save`, `dashboard.title`).
Ortak buton/etiketler `common` altında (bir kez çevrilir).
